Важно
-----
1. Для работы необходим Content Monster 2 Tree Edit версии не ниже 1.1.5
2. Для установки разверните архив с плагином в папку CM2 /plugins
3. Файлы словарей должны быть в кодировке UTF-8 без Byte Order Mask(BOM)
4. Внимательно смотрите за тем, что-бы текстовый разделитель и разделитель позиций совпадали при экспорте в CSV и настройках плагина.
5. Пример шаблона позиции: {$1}
6. Шаблоны позиций нумеруются с 1
7. Пример шаблона случайного текста: {$Строка один|Строка 2|Ещё строка}
8. Пример шаблона для числа прописью: {$1:s}
9. Пример шаблона указания словаря для значения: {$1:fingers.yml}
   * расширение ".yml" должно присутствовать, как метка использования словаря

Пример шаблона
--------------

{$Стоимость часа|Цена|Стоимость}: {$2:s} {$рублей/час|руб./час}
{$Время работы|Часы|Часы работы}: {$3}

Тип парной: {$11}
Описание: {$4}, {$5:fingers.yml}, {$6}.

Телефон: {$7}
Адрес: {$8}
Ближайшая станция метро: {$9}
{$В наличии|Есть}: {$10}

Фотографии:
<img src="/images/{$12}"> <img src="/images/{$13}"> <img src="/images/{$14}"> 

Пример словаря
--------------

!!python/tuple [0,50.9]:
- "%s кг, что достаточно мало"
- "%s кг, это вообщем скромно"
!!python/tuple [51,99]:
- "%s кг, что удовлетворительно"
- "%s кг, что обычно для этого продукта"
!!python/tuple [100,1000]:
- "более ста килограмм"
- "%s кг, что довольно много"

!!python/unicode "зеленый":
- "%s"
- "светлозеленый"
!!python/unicode "красный":
- "%s"
- "пурпурный"

%s - может быть в тексте, тогда на его место подставляется исходная фраза, либо его может не быть.
